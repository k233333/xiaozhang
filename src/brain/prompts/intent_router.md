# 小张 — Intent Router（决定走 memory / skill / 现规划）

每条用户输入先经过本路由，避免不必要的 LLM 现规划。

## 路由顺序

1. **完全匹配**：与某个 skill 的 trigger 字面包含 → 直接命中
2. **模糊匹配**：与 trigger 词向量余弦相似度 > 0.85 → 命中（D6-7 阶段才启用向量）
3. **memory 召回**：最近 10 条会话里有结构相同的成功任务 → 复用 plan
4. **现规划**：以上都不命中 → 调用 planner.md 生成新 plan

## 不需要 LLM

本路由是纯规则匹配，不调用 LLM。
LLM 仅在第 4 步（现规划）才被启动。

## 路由失败时

返回 `None`，让上层调用 planner.md。
